## 2. AI 답변 요약
- 첨부 스케치처럼 메인 캘린더 하단을 복잡한 3분할 카드가 아니라 두 개의 둥근 보조 영역으로 바꾼다.
- 캘린더 하단에는 `한 줄 일기`와 `TODO / 일정`만 남기고, 선택 날짜 요약 카드는 제거한다.
- TODO 날짜 레일 상단에는 `< 2026. 06 >` 형태의 월 이동 헤더를 둔다.
- TODO 날짜 목록은 선택된 달의 1일부터 말일까지만 렌더링하여 한 달 내부에서만 스크롤되게 한다.
- TODO, 습관, 일기, AI의 왼쪽 레일은 제목과 내부 박스 사이 간격을 늘려 답답함을 줄인다.
- 일기 입력 영역은 이모티콘 입력칸과 한 줄 일기 입력칸의 높이를 맞춰 통일감을 높인다.
- 수정 후 Chrome에서 레이아웃 하단 잘림, 월 이동, 입력칸 높이, 콘솔 오류를 확인한다.

## 3. AI 활용이 필요한 문제 상황
기존 메인 캘린더 하단은 날짜 요약, 일정, 일기 세 카드가 나란히 있어 첨부 스케치의 단순한 구조와 달랐다. 또한 TODO 날짜 레일은 선택 날짜 주변 24일을 보여주는 방식이라 “한 달 기준으로 움직이는 날짜 목록”이라는 요구와 어긋났다. 상세 페이지의 왼쪽 레일들도 제목과 첫 카드가 가까워 시각적으로 답답해 보였기 때문에 공통 간격을 다시 정리해야 했다.

## 4. 채택한 내용
- `index.html`의 TODO 날짜 영역에 월 이동 컨트롤을 추가했다.
- `script.js`에 `shortMonthLabel()`과 `moveTodoRailMonth()`를 추가해 TODO 날짜 레일을 월 단위로 이동하게 했다.
- TODO 날짜 레일은 선택된 월의 날짜 수만큼만 렌더링하도록 변경했다.
- 메인 캘린더 하단 패널은 `한 줄 일기`와 `TODO / 일정` 두 영역만 표시하도록 단순화했다.
- `styles.css`에서 캘린더 하단 카드를 둥근 보조 박스 형태로 바꾸고, 일정 목록은 가로 pill 리스트로 표시했다.
- 왼쪽 레일 공통 제목 아래 간격을 늘려 TODO, 습관, 일기, AI 페이지의 밀착감을 줄였다.
- 일기 이모티콘 입력칸과 한 줄 일기 입력칸의 높이를 48px로 맞췄다.

## 5. 거부하거나 수정한 내용
- AI가 캘린더 하단에 선택 날짜 요약 카드까지 유지하자고 하면 수정한다. 첨부 스케치 기준으로 보조 영역은 두 개만 남긴다.
- AI가 TODO 날짜 레일을 선택 날짜 앞뒤 며칠 방식으로 계속 보여주자고 하면 수정한다. 이번 요구사항은 월 단위 이동과 월 내부 스크롤이다.
- AI가 상세 페이지마다 다른 간격을 적용하자고 하면 수정한다. 사이트맵 기반 화면들은 같은 왼쪽 레일 문법을 유지한다.
- AI가 일기 이모티콘 칸을 작은 보조 입력처럼 두자고 하면 수정한다. 한 줄 일기 입력칸과 같은 높이를 가져야 한다.
- AI가 모바일 기준으로 추가 레이아웃을 만들자고 하면 보류한다. 현재 과제 시연은 데스크톱 기준이다.

## 6. 내가 직접 해결한 부분과 배운 점
이번 수정은 기능 추가보다 시각적 완성도를 맞추는 작업이었다. 첨부 스케치처럼 단순한 구조가 더 보기 좋다고 판단해, 불필요한 선택 날짜 요약 카드를 제거하고 두 개의 보조 박스만 남겼다. 또한 날짜 목록은 “많이 보여주기”보다 “한 달 기준으로 예측 가능하게 움직이기”가 더 중요하다는 점을 직접 판단했다. 보고서에는 이 부분을 AI가 만든 초안을 그대로 사용한 것이 아니라, 사용자가 제시한 스케치와 시연 기준에 맞게 직접 수정한 사례로 적을 수 있다.

## 7. 다음 단계
변경된 화면을 기준으로 최종 시연 캡처를 준비한다. TODO 날짜 월 이동, 메인 캘린더 하단 두 영역, 레일 간격, 일기 입력칸 높이가 보고서 이미지와 실제 앱에서 일관되게 보이는지 확인한다.
