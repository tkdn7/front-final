# 23 TODO습관일기 정렬월이동수정

## 1. 사용자 프롬프트 원문
[메인페이지]
1. TODAY TODO 리스트에 있는 TODO 리스트 페이지로 이동할 수 있는 아이콘 버튼이 navigation의 TODO 아이콘 이미지와 다름
-> TODAY TODO 리스트의 TODO 리스트 페이지로 이동할 수 있는 아이콘 버튼 navigation의 TODO 아이콘 이미지와 동일하도록 수정
[TODO]
1. 좌측의 TODO 날짜에 디테일한 날짜(2026년 6월 1일(월))까지 표기할 필요 없음
-> 디테일한 날짜 없애고, 미완료 항목 개수가 더 잘 보이도록 수정. 미완료 몇개인지 알려주는 글씨 크기 조금 확대.
2. TODO 일정관리에 일정추가 버튼 오른쪽 끝나는 지점이 아래 일정 항목 박스와 동일한 위치에서 끝나지 않음.
-> 일정추가 버튼의 우측 부근이 일정 표시되는 박스들의 우측 부근과 동일한 지점에 위치하도록 일정 추가 버튼 좌측으로 조금 이동
[습관]
1. 습관관리의 습관추가 버튼이 TODO에서 말했던 것과 동일하게 습관관리 항목들 박스가 끝나는 우측 지점보다 더 튀어나오는 문제가 있음
-> 습관 이름 적는 칸 크기를 줄여 습관 추가 버튼이 끝나는 우측 지점과 습관관리 항목들 박스가 끝나는 우측 지점이 동일해지도록 수정
[일기]
1. 일기 페이지의 좌측 일기날짜 또한 TODO 리스트의 TODO 날짜 처럼 달마다 이동할 수 있도록 만들어져야 함

## 2. AI 답변 요약
- 메인 TODAY TODO 이동 아이콘을 navigation의 TODO 아이콘과 같은 `todo-icon.png`로 통일한다.
- TODO 날짜 레일에서는 긴 날짜 문장을 제거하고 날짜 숫자와 미완료 개수만 보이게 한다.
- 미완료 개수 텍스트는 더 크게 보이도록 별도 클래스를 적용한다.
- TODO 일정 추가 버튼은 리스트 항목 오른쪽 끝과 맞도록 오른쪽 여백을 둔다.
- 습관 추가 폼은 이름 입력칸 폭을 줄이고 폼 오른쪽 여백을 둬 항목 박스와 끝선을 맞춘다.
- 일기 날짜 레일에는 TODO 날짜와 같은 월 이동 컨트롤을 추가한다.
- 일기 날짜 목록도 해당 월의 날짜만 표시되도록 변경한다.
- 브라우저 검증에서 메인 TODO 아이콘과 navigation TODO 아이콘이 같은 파일로 확인됐다.

## 3. AI 활용이 필요한 문제 상황
메인, TODO, 습관, 일기 페이지가 각각 비슷한 구조를 사용하지만 아이콘, 날짜 레일, 버튼 끝선이 조금씩 달라 통일성이 떨어졌다. 특히 일기 날짜는 TODO처럼 월 단위로 이동할 수 없어 같은 사이트맵 구조를 따르지 못했다.

## 4. 채택한 내용
- 메인 TODAY TODO 아이콘을 `add-todo-icon.png`에서 `todo-icon.png`로 변경했다.
- TODO 날짜 레일의 긴 `koreanDate` 문구를 제거하고 `미완료 n개` 중심으로 표시했다.
- `.mini-date-count`를 추가해 미완료 개수 글씨를 키우고 굵게 보이도록 했다.
- `#openTodoDialog`에 오른쪽 여백을 적용해 일정 항목 박스와 끝선을 맞췄다.
- `#habitForm`에서 습관 이름 입력칸 폭을 줄이고 오른쪽 여백을 추가했다.
- 일기 좌측 레일에 `diaryRailPrev`, `diaryRailNext`, `diaryRailMonthLabel`을 추가했다.
- `moveDiaryRailMonth()`를 만들어 일기 페이지도 TODO처럼 월 단위로 이동하도록 했다.
- Playwright 검증에서 TODO 버튼과 일정 박스 오른쪽 끝 차이 0px, 습관 추가 버튼과 습관 박스 오른쪽 끝 차이 0px로 확인했다.
- 일기 레일은 2026. 06에서 2026. 07로 이동하고 날짜 개수도 30개에서 31개로 바뀌는 것을 확인했다.

## 5. 거부하거나 수정한 내용
- TODO 날짜 레일에 긴 날짜를 계속 표시하는 방식은 공간 낭비가 커서 제거했다.
- 버튼을 항목 박스보다 더 오른쪽으로 튀어나오게 두는 방식은 정렬감을 해치므로 수정했다.
- 일기 날짜를 선택일 주변 24일만 보여주는 기존 방식은 TODO 날짜 구조와 다르므로 월 단위 방식으로 변경했다.

## 6. 내가 직접 해결한 부분과 배운 점
이번 단계에서는 같은 기능군의 UI가 같은 패턴을 가져야 사용자가 화면을 예측하기 쉽다는 점을 확인했다. 아이콘 하나, 버튼 끝선 하나, 날짜 레일 방식 하나가 달라도 전체 디자인 통일성이 떨어질 수 있으므로 반복 구조는 가능한 같은 규칙으로 맞추는 것이 중요하다.

## 7. 다음 단계
실제 화면에서 아이콘 통일, 날짜 레일 가독성, 버튼 끝선 정렬, 일기 월 이동이 자연스럽게 보이는지 확인한다. 이후 수정이 필요하면 각 페이지의 공통 레일/정렬 규칙을 유지한 상태에서 간격만 미세 조정한다.
