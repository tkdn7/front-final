# 18 BASIC기준 캘린더비율복원

## 1. 사용자 프롬프트 원문
캘린더 2분할 하단은 그대로 유지해.
다만, 아무것도 선택하기 전 캘린더와 선택 후 2분할 비율은 github의 BASIC과 동일하도록 수정

commit & push 이름을 BASIC으로 보냈어

## 2. AI 답변 요약
- GitHub에서 `BASIC`이라는 이름은 브랜치가 아니라 커밋 메시지로 확인했다.
- `BASIC` 커밋의 `styles.css`를 기준으로 캘린더 외부 비율 관련 CSS를 다시 비교했다.
- 현재 하단 한줄일기/TODO 카드 내부 구성은 유지한다.
- 캘린더 그리드를 강제로 flex로 늘리던 설정은 제거한다.
- 날짜 칸은 `BASIC`처럼 `min-height: 42px` 기준으로 되돌린다.
- 선택 후 나오는 하단 2분할 영역은 `flex: 0 0 auto`로 바꿔 BASIC과 같은 높이 흐름을 따른다.
- 하단 카드의 제목, 스크롤, 내부 비율 스타일은 이전 수정안을 유지한다.
- 브라우저 검증에서 선택 전 캘린더 240px, 선택 후 하단 189px, 날짜칸 42px로 BASIC 흐름을 확인했다.

## 3. AI 활용이 필요한 문제 상황
이전 단계에서 캘린더와 하단 영역을 1:1 비율로 맞췄지만, 사용자가 GitHub에 올린 `BASIC` 상태를 기준으로 선택 전/후 캘린더 비율을 맞추길 원했다. 따라서 최신 요구사항을 반영하려면 로컬 코드만 보고 판단하지 않고, GitHub의 `BASIC` 커밋과 현재 CSS를 비교해야 했다.

## 4. 채택한 내용
- GitHub `BASIC` 커밋 `4bdc98f`의 `styles.css`를 기준 자료로 사용했다.
- `.calendar-grid`에서 `flex`, `min-height`, `grid-auto-rows`를 제거했다.
- `.day-cell`은 `BASIC`과 동일하게 `min-height: 42px`을 사용하도록 수정했다.
- `.date-split`은 `flex: 0 0 auto`로 바꿔 선택 후 하단 영역이 캘린더를 과하게 밀어내지 않게 했다.
- `.date-split.active`에서 강제 flex 확장 값을 제거했다.
- Playwright로 선택 전/후를 확인했고, 하단 영역이 캘린더 패널 밖으로 넘치지 않는 것을 확인했다.

## 5. 거부하거나 수정한 내용
- 하단 한줄일기/TODO 카드 내부 UI까지 `BASIC`으로 되돌리는 것은 거부했다. 사용자는 하단은 그대로 유지하라고 했기 때문이다.
- 이전 단계의 1:1 고정 비율은 최신 요구사항과 충돌하므로 수정했다.
- GitHub에서 `BASIC` 브랜치를 찾는 방식은 맞지 않아 커밋 메시지 기준으로 다시 확인했다.
- 캘린더와 하단 영역 모두를 무조건 같은 높이로 고정하는 방식은 BASIC 기준과 다르므로 제외했다.

## 6. 내가 직접 해결한 부분과 배운 점
GitHub 기준 버전을 다시 확인하면서, 같은 화면이라도 내부 카드 디자인과 외부 비율 규칙은 분리해서 수정해야 한다는 점을 확인했다. 이번에는 사용자의 요구를 반영해 하단 카드 디자인은 유지하고, 캘린더가 선택 전/후에 차지하는 높이 흐름만 `BASIC` 커밋과 맞췄다.

## 7. 다음 단계
사용자가 실제 화면을 보며 BASIC 기준 비율이 의도한 느낌과 맞는지 최종 확인한다. 이후 추가 수정이 필요하면 하단 카드 내부가 아니라 캘린더 외부 높이 규칙만 우선 조정한다.
