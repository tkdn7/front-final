# TODO 모달, 일기 기본 이모티콘, AI 브리핑 대상 수정

## 1. 사용자 프롬프트 원문
[TODO]

2. 일정 추가 버튼 클릭시에 나오는 일정추가버튼 가로 길이 늘려줘. 가로 시작하는 좌측 위치는 동일하게 유지하고, 우측 끝나는 위치가 시간 선택 박스 우측과 동일한 위치에 끝나도록 수정해줘. 
[일기]
1. 일기 추가 시에 이모티콘 기본적으로 🙂가 들어가게 해줘. 이모티콘 작성되어 있지 않을 때 기본 이모티콘이 들어가게 하고, 경고창은 없애줘. 물론, 사용자가 원하는 이모티콘으로 바꿀 수 있는 기능은 유지되어야 해.
[AI]
1. 좌측의 브리핑 대상에서 요약에 나오는 TODO 미완료 개수가 일치하지 않는 문제가 있어. 다시 한번 확인해줘.
2. 좌측의 브리핑 대상에서 선택 가능 일정의 항목 표기 순서는 시간의 오름차순으로 표시되도록 수정해줘.

## 2. AI 답변 요약
- TODO 일정 추가 모달에서 지역 라벨 옆의 `선택` 텍스트를 제거했다.
- TODO 모달의 일정 추가 버튼은 폼 전체 폭을 사용하도록 해, 좌측 시작점은 유지하고 우측 끝은 시간 입력 박스 우측과 맞췄다.
- 일기 모달의 이모티콘 입력값은 기본적으로 🙂가 들어가도록 되돌렸다.
- 이모티콘을 비운 상태로 저장하면 경고창을 띄우지 않고 🙂를 기본값으로 저장하도록 수정했다.
- AI 브리핑 대상 요약의 TODO 미완료 개수는 오늘 TODO가 아니라 전체 미완료 TODO 기준으로 계산하도록 바꿨다.
- AI의 선택 가능 일정 목록은 우선순위가 아니라 시간 오름차순으로 표시되도록 별도 정렬 함수를 추가했다.
- 로컬 Chrome 검증으로 버튼 폭, 기본 이모티콘 저장, AI 미완료 개수, 일정 시간 정렬을 확인했다.

## 3. AI 활용이 필요한 문제 상황
이번 수정은 UI 문구와 버튼 폭처럼 작은 시각 요소뿐 아니라, AI 브리핑 대상의 데이터 기준도 함께 수정해야 했다. 특히 TODO 미완료 개수가 메인 화면 기준과 AI 요약 기준에서 다르게 계산되면 보고서와 시연에서 혼란이 생길 수 있어 기준을 통일해야 했다.

## 4. 채택한 내용
- TODO 모달 지역 라벨은 `지역`만 보이도록 단순화했다.
- `#todoForm .primary-button`에만 폼 전체 폭 스타일을 적용해 다른 버튼에는 영향이 없도록 했다.
- 일기 이모티콘은 사용자가 수정할 수 있지만, 비워두면 🙂가 저장되도록 했다.
- AI 요약의 TODO 미완료 개수는 `pendingTodos()` 전체 길이를 기준으로 표시했다.
- AI 선택 가능 일정은 `time` 값을 기준으로 오름차순 정렬해 보여주도록 했다.

## 5. 거부하거나 수정한 내용
- 이모티콘 미입력 경고창은 사용자 요청에 따라 제거했다.
- AI 요약에서 오늘 TODO만 세는 방식은 메인 SCHEDULE의 전체 미완료 기준과 맞지 않아 수정했다.
- 선택 가능 일정을 우선순위 기준으로 보여주는 방식은 이번 요구와 맞지 않아 AI 영역에서는 시간 기준 정렬을 별도로 적용했다.
- 전체 버튼 스타일을 바꾸면 다른 페이지 버튼까지 흔들릴 수 있어 TODO 모달 버튼에만 제한적으로 적용했다.

## 6. 내가 직접 해결한 부분과 배운 점
- 같은 TODO 데이터라도 화면 목적에 따라 정렬 기준이 달라질 수 있음을 확인했다. 메인 목록은 날짜와 우선순위가 중요하지만, AI 브리핑 대상 목록은 하루 일정 흐름을 보기 위해 시간순이 더 적절했다.
- 기본 이모티콘은 사용자의 입력 부담을 줄이는 방향이라 경고창보다 자연스러운 UX가 될 수 있다고 판단했다.
- 버튼 폭처럼 작은 UI 수정도 실제 브라우저 좌표로 확인해야 디자인 의도와 맞는지 정확히 알 수 있었다.
