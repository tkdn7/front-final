# 12 TODO 일정 통합 상세수정

## 1. 사용자 프롬프트 원문
A. 메인 페이지
	1. 전체 페이지의 각 레이아웃 스크롤 버튼이 다른 요소와 붙어있지 않도록 수정
	2. 캘린더 내부의 일정 및 한 줄 일기 칸이 눌렀을때 animation 효과로 나와야하지 처음부터 나와있으면 안됨 추가로 너무 크게 나오기 때문에 2분할을 할 때 비율이 캘린더와 1:1일 필요없음 추가로 한 줄 일기와 일정은 세로로 ㅁ, ㅁ이 아닌 가로로 순서대로 배치되도록 변경
	3. TODO 리스트에서 완료한 작업은 블러 처리해서 가독성 향상
	4. TODO와 일정은 같은 기능이기 때문에 TODO에서 나오는 것이 캘린더의 날짜를 클릭했을때 연동되서 나와야함
	5. 습관도 완료한 것은 블러처리로 가독성 향상

B. TODO 페이지
	1. TODO 페이지의 TODO 관리 레이아웃을 사이트맵.pdf처럼 세로 구성의 미니 캘린더로 변경하고 날짜 옆에는 수행되지 않은 일정의 갯수 표시 및 스크롤을 통해서 날짜들을 확인할 수 있도록 수정
	2. 우측의 추가한 순서 레이아웃 내부의 최우측 갯수 표시 대신 일정 추가 버튼을 두고 클릭시 모달 팝업으로 일정을 추가할 수 있도록 수정
	3. 추가로 추가한 순서 레이아웃 내부 좌측 상단의 ORDER, 추가한 순서는 각각 TODO, 일정관리로 변경
	4. 메인의 TODO와 동일하게 완료한 일정은 블러 처리
	5. TODO 페이지에서 보일때 우선순위 > 남은 기한 > 추가한 순서 순으로 변경

C. 습관 페이지
	1. 오늘 해야할 습관이 아닌 최고(연속 수행 일자) 연습 습관 순위 1~3위와 한 달동안 평균 달성률(모든 습관의 평균), %로 표시하도록 변경
	2. 습관 관리 페이지에서 습관 레이아웃의 오늘 완료 (점) 이번 달 1회 (점) 몇 퍼센트인지 % 표기하도록 수정
	3. 습관 관리 레이아웃 내부의 최우측 최상단 6개 표시 삭제

D. 일기 페이지
	1. 날짜별 한 줄 일기에서 한 줄 일기 적을때 칸 조정 불가능하도록 변경 (어차피 한 줄이기 때문에 칸이 조정하면서 넓힐 필요가 없음) 
	2. 좌측 일기 날짜를 위에서 얘기했던 TODO 페이지의 좌측 레이아웃과 유사하도록 세로 캘린더 및 스크롤로 변경
	3. 날짜 옆에는 한 줄 일기를 적을때 넣었던 이모티콘으로 수정
	4. 일기 페이지에서는 일정 관련된 내용이 보이지 않도록 수정

D. AI 페이지
	1. 좌측의 브리핑 대상 레이아웃 내에서 스케줄에서 브리핑 대상 일정 내에서 선택 가능하도록 변경
	2. 브리핑 대상의 내부의 레이아웃의 제목칸과 박스들이 조금씩 더 띄워서 가독성 향상하도록 수정

## 2. AI 답변 요약
- 메인 캘린더의 선택 날짜 상세 패널은 기본 숨김 상태로 두고, 날짜 클릭 시 애니메이션으로 나타나게 수정한다.
- TODO와 일정은 별도 데이터처럼 보이지 않도록 TODO의 dueDate를 캘린더 일정으로 함께 사용한다.
- 완료한 TODO와 습관은 흐림 처리와 취소선으로 완료 상태를 구분한다.
- TODO 상세 화면은 사이트맵처럼 왼쪽 세로 날짜 레일, 오른쪽 일정관리 목록 구조로 바꾼다.
- 일정 추가는 우측 상단 버튼에서 모달 팝업으로 처리한다.
- 습관 상세의 왼쪽 영역은 오늘 할 일 목록이 아니라 랭킹과 한 달 평균 달성률로 바꾼다.
- 일기 상세 화면에서는 일정 관련 영역을 제거하고, 날짜 레일과 한 줄 일기 입력만 남긴다.
- AI 상세 화면의 왼쪽 영역에서 브리핑 대상 일정을 직접 선택할 수 있게 한다.

## 3. AI 활용이 필요한 문제 상황
이전 단계에서 전체 기능은 들어갔지만, TODO와 일정이 분리되어 보여 사용자가 같은 의미의 정보를 두 번 관리하는 것처럼 보였다. 또한 메인 캘린더 상세 정보가 처음부터 보이거나 너무 크게 보여 사이트맵의 간단한 구조와 맞지 않았다. 상세 화면도 TODO, 습관, 일기, AI가 모두 같은 화면 문법을 갖도록 다시 정리할 필요가 있었다.

## 4. 채택한 내용
- `script.js`에서 TODO의 마감 날짜를 캘린더 일정으로 해석하는 `scheduleItemsForDate()` 흐름을 사용했다.
- 메인 캘린더 상세 패널은 `datePanelOpen` 상태를 두어 클릭 전에는 숨기고, 클릭 후 `revealPanel` 애니메이션으로 열리게 했다.
- 선택 날짜의 일정과 한 줄 일기는 세로 카드가 아니라 가로 배치 카드로 정리했다.
- TODO 상세 화면의 왼쪽은 24일 범위의 세로 미니 캘린더로 만들고, 날짜별 미완료 개수를 표시했다.
- TODO 추가 폼은 화면에 항상 두지 않고 `일정 추가` 버튼을 누르면 모달로 열리게 했다.
- TODO 목록 정렬은 우선순위, 남은 기한, 추가된 순서 기준으로 바꿨다.
- 습관 상세 왼쪽에는 한 달 평균 달성률과 연속 수행 순위 1~3위를 배치했다.
- 일기 상세에서는 일정 관련 요소를 제거하고, 날짜 옆에는 저장된 이모티콘이 보이도록 했다.
- AI 상세 왼쪽의 브리핑 대상 목록을 버튼형 일정 선택 영역으로 바꾸고, 선택 상태를 표시했다.

## 5. 거부하거나 수정한 내용
- AI가 TODO와 일정을 별도 기능으로 계속 유지하자고 하면 수정한다. 이번 프로젝트에서는 TODO의 마감 날짜가 곧 캘린더 일정 역할을 한다.
- AI가 캘린더 상세 정보를 처음부터 펼쳐 두는 방식을 제안하면 거부한다. 사이트맵처럼 메인 달력을 먼저 보이고, 클릭 후 보조 정보가 나와야 한다.
- AI가 선택 날짜 상세 영역을 캘린더와 1:1 크기로 크게 잡으면 수정한다. 메인 화면이 잘리지 않도록 보조 정보 영역은 작고 가로형으로 유지한다.
- AI가 TODO 추가 폼을 상세 화면 상단에 고정하자고 하면 수정한다. 사이트맵의 단순한 목록 구조를 유지하기 위해 모달 팝업으로 분리한다.
- AI가 일기 화면에 일정 목록을 다시 넣자고 하면 제거한다. 일기 화면은 한 줄 일기와 날짜 선택에만 집중한다.
- AI가 습관 왼쪽 영역에 오늘 할 습관 목록을 다시 두자고 하면 수정한다. 사용자가 요청한 랭킹과 평균 달성률을 우선한다.

## 6. 내가 직접 해결한 부분과 배운 점
이번 단계에서는 기능을 더 추가하는 것보다, 같은 기능이 중복되어 보이지 않도록 의미를 정리하는 일이 중요했다. TODO와 일정을 통합해서 생각해야 캘린더, AI 브리핑, TODO 페이지가 하나의 데이터 흐름으로 연결된다는 점을 직접 판단했다. 또한 화면이 잘리지 않게 하려면 전체 페이지 스크롤보다 각 패널 내부 스크롤, 숨김/펼침 상태, 가로 배치 비율을 함께 조정해야 한다는 점을 배웠다.

## 7. 다음 단계
