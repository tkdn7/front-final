# 11 사이트맵 재정렬 AI질문

## 1. 사용자 프롬프트 원문
1. AI 브리핑
1) 버튼 레이아웃 조정이 더 필요할 것 같음 ex) 메인의 오늘 조언 생성이 실제 조언창과 너무 붙어있음
2) AI 브리핑의 수준이 아쉬움
조금 더 깔끔하게 답변을 내놓을 수 있도록 프롬프트 수정안을 내놓고 어떻게 수정하면 될지 알려줘
2. 현재 메인창이 스크롤을 안하면 페이지 기본 확대 수준에서 짤려서 보이기 때문에 수정 필요
3. 습관, 일기, AI의 페이지의 UI 통일성을 위해서 사이트맵.pdf를 다시 한 번 확인하고 수정 필요
4. 전체적으로 사이트맵의 디자인을 벗어난 경우들이 보임 이 또한 사이트맵.pdf를 조금 더 성실하게 따르는 것으로 수정
5. AI 페이지에서 자신의 일정에 대한 질문을 할 수 있도록 수정할 것
6. 메인 페이지의 캘린더 내부 디자인이 마음에 들지 않음 날짜별로 네모칸으로 나누는 것을 자제할 것

## 2. AI 답변 요약
- 사이트맵 PDF를 다시 렌더링하여 메인과 상세 화면 구조를 재확인했다.
- 메인 캘린더는 날짜별 네모 카드가 아니라 텍스트 중심 달력으로 수정한다.
- 습관, 일기, AI 화면은 왼쪽 보조 영역과 오른쪽 주요 영역을 가진 공통 레이아웃으로 맞춘다.
- 메인 화면은 페이지 전체 스크롤 없이 기본 확대 상태에서 보이도록 높이와 내부 스크롤을 조정한다.
- AI 브리핑 버튼과 답변 영역 사이에 간격을 추가한다.
- AI 페이지에 일정 질문 입력창을 추가하고, 질문이 있으면 질문 답변용 프롬프트 형식을 사용한다.
- AI 응답은 정해진 섹션 형식으로 짧고 구체적으로 나오게 프롬프트를 수정한다.

## 3. AI 활용이 필요한 문제 상황
초기 구현은 기능은 들어갔지만, 사이트맵의 손그림 구조와 비교했을 때 상세 화면의 배치가 서로 다르고 메인 캘린더가 과하게 카드형으로 보였다. 또한 AI 답변이 자유 형식이라 보고서 캡처에 넣었을 때 정돈된 인상을 주기 어려웠다. 따라서 사이트맵을 다시 기준으로 삼아 UI와 프롬프트를 함께 조정해야 했다.

## 4. 채택한 내용
- `사이트맵.pdf`를 다시 확인하고 메인/상세 화면 구조를 재정렬했다.
- 메인 캘린더의 날짜 박스 테두리를 제거하고 숫자와 점 중심으로 바꿨다.
- 습관, 일기, AI 상세 화면을 `왼쪽 목록 + 오른쪽 본문` 구조로 통일했다.
- AI 페이지에 “내 일정 질문” 입력창을 추가했다.
- AI 브리핑 프롬프트에 출력 형식을 명확히 지정했다.
- 메인 화면은 `100vh` 기준으로 맞추고 각 패널 내부가 스크롤되도록 조정했다.

## 5. 거부하거나 수정한 내용
- AI가 사이트맵과 상관없는 새로운 대시보드 디자인을 제안하면 거부한다.
- AI가 긴 설명형 답변을 자유롭게 출력하게 두면 수정한다. 화면에는 짧은 섹션형 답변이 더 적합하다.
- AI가 모바일 반응형을 다시 우선하자고 하면 수정한다. 현재 테스트 기준은 데스크톱이다.
- AI가 캘린더 날짜를 모두 카드처럼 강조하면 수정한다. 사이트맵은 간단한 달력 숫자 배열에 가깝다.
- AI가 질문 기능을 별도 페이지로 분리하자고 하면 수정한다. 사이트맵의 AI 화면 안에 포함하는 것이 더 맞다.

## 6. 내가 직접 해결한 부분과 배운 점
사이트맵을 단순 참고 자료로 두면 구현이 점점 다른 방향으로 갈 수 있다는 점을 배웠다. 그래서 이번 단계에서는 PDF를 다시 확인한 뒤, 캘린더 모양, 상세 화면 공통 레이아웃, AI 질문 위치를 사이트맵 기준으로 직접 조정했다. 또한 AI 답변의 품질은 모델만의 문제가 아니라 프롬프트에서 출력 형식과 길이를 얼마나 명확히 제한하는지에 크게 영향을 받는다는 점을 확인했다.

## 7. 다음 단계
데스크톱 Chrome에서 메인 화면이 잘리지 않는지, 캘린더가 네모 카드처럼 보이지 않는지, 습관/일기/AI 화면이 같은 구조로 보이는지, AI 질문 입력창과 API 모달 흐름이 정상 동작하는지 검증한다.
