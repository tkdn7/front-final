# 메인 캘린더 미완료 일정 점 표시 수정

## 1. 사용자 프롬프트 원문
[메인페이지]
1. 캘린더에 점 표시는 미완료 일정만 표시되어야 해.
-> 완료된 작업은 메인페이지 캘린더에서 점 표시에 포함되지 않도록 수정 (캘린더 하단의 TODO/일정에는 지금과 동일하게 블러처리된 상태로 표시되어야 함)

## 2. AI 답변 요약
- 메인 캘린더 날짜 칸의 점 표시 계산에서 완료된 TODO를 제외하도록 수정했다.
- 캘린더 하단 TODO/일정 패널은 기존처럼 완료 항목까지 표시되도록 유지했다.
- 완료 항목은 하단 패널에서 `done-item` 클래스로 블러/완료 처리된 상태를 유지한다.
- 로컬 Chrome 검증에서 미완료 2개, 완료 1개 데이터를 넣어 점은 2개만 표시되는지 확인했다.
- 같은 검증에서 하단 TODO/일정 목록은 3개가 모두 표시되고, 완료 항목 1개가 블러 처리되는 것을 확인했다.

## 3. AI 활용이 필요한 문제 상황
캘린더의 점 표시는 일정 존재 여부를 빠르게 보여주는 요약 정보이고, 하단 TODO/일정 패널은 선택한 날짜의 실제 항목 전체를 보여주는 상세 정보다. 따라서 같은 데이터를 사용하더라도 요약과 상세의 표시 기준을 다르게 적용해야 했다.

## 4. 채택한 내용
- `renderCalendar()`에서 점을 만들 때 `dayEvents.filter((event) => !event.done)`을 적용했다.
- `renderSelectedDatePanel()`은 수정하지 않아 완료 항목도 하단 목록에 남도록 했다.
- 기존 우선순위 색상 정렬은 미완료 항목 안에서 그대로 유지했다.

## 5. 거부하거나 수정한 내용
- 완료 항목을 전체 일정 데이터에서 제거하는 방식은 하단 TODO/일정 목록까지 사라지므로 제외했다.
- 하단 패널의 완료 항목 표시 방식을 바꾸는 것도 사용자 요청과 달라서 수정하지 않았다.

## 6. 내가 직접 해결한 부분과 배운 점
- 같은 날짜 데이터를 점 표시와 상세 목록에 각각 다르게 쓰는 것이 더 명확한 UX라는 점을 확인했다.
- 요약 UI에서는 현재 해야 할 일, 즉 미완료 항목만 강조하고, 상세 UI에서는 완료 기록까지 유지하는 방식이 적절했다.
