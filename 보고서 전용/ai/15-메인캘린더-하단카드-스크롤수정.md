# 15 메인캘린더 하단카드 스크롤수정

## 1. 사용자 프롬프트 원문
[메인페이지] - 사진참고
1. 캘린더 2분할 한줄일기와 TODO/일정 하단 칸까지 모두 채워도 됨.
다만, 한줄일기와 TODO/일정에서 비율은 한줄일기가 더 넓지 않도록 주의.
2. 한줄일기 내용이 긴 경우 TODO/일정과 동일하게 스크롤해서 볼 수 있도록 수정.
3. 메인페이지의 다른칸과 동일하게 한줄일기와 TODO/일정 소제목은 해당 칸의 상단에 작성되며, 그의 세부내용이 아래 작성되도록 수정
4. 한줄일기 이모티콘 사이즈 키우기

## 2. AI 답변 요약
- 메인 캘린더 하단 영역을 남는 높이까지 채우도록 `date-split`을 flex 확장 구조로 바꾼다.
- `한줄일기` 카드가 `TODO / 일정` 카드보다 커지지 않도록 grid row 비율을 조정한다.
- 각 카드의 제목은 상단 헤더에 고정하고, 세부 내용은 아래 본문 영역에 배치한다.
- 한줄일기 본문은 긴 내용이 들어와도 카드가 늘어나지 않고 내부 스크롤되도록 고정한다.
- TODO/일정 목록도 카드 내부에서 세로 스크롤되도록 유지한다.
- 한줄일기 이모티콘은 별도 큰 영역으로 분리해 시각적으로 더 크게 보이게 한다.
- Chrome에서 긴 일기 텍스트를 넣어 실제 내부 스크롤이 생기는지 검증한다.

## 3. AI 활용이 필요한 문제 상황
이전 메인 캘린더 하단은 한줄일기와 TODO/일정이 한 줄 안에 압축되어 보였고, 제목과 내용의 위계가 메인 페이지의 다른 카드들과 달랐다. 또한 긴 한줄일기가 들어오면 카드 안에서 읽기 어렵거나 레이아웃이 늘어날 가능성이 있었다. 첨부 스케치처럼 제목은 상단, 내용은 하단에 배치하고 내부 스크롤을 적용해야 했다.

## 4. 채택한 내용
- `script.js`의 선택 날짜 패널 마크업을 `split-card-header`, `diary-card-body`, `schedule-pill-list` 구조로 변경했다.
- 한줄일기 카드 상단에는 `한줄일기`와 날짜를, 본문에는 큰 이모티콘과 일기 텍스트를 배치했다.
- TODO/일정 카드 상단에는 `TODO / 일정`과 일정 개수를, 본문에는 일정 버튼 목록을 배치했다.
- `styles.css`에서 `date-split`을 남은 하단 영역까지 확장되도록 `flex: 1 1 0`으로 조정했다.
- 한줄일기와 TODO/일정의 높이 비율을 `0.82fr : 1.18fr`로 잡아 한줄일기가 더 커지지 않도록 했다.
- 한줄일기 본문에는 `overflow: auto`, 고정 높이, grid row 보정을 적용해 긴 내용이 내부 스크롤되게 했다.
- 이모티콘 크기는 `46px`로 키워 스케치처럼 눈에 띄게 만들었다.

## 5. 거부하거나 수정한 내용
- AI가 한줄일기 카드를 TODO/일정보다 더 크게 잡는 제안은 수정한다. 사용자 요청상 한줄일기가 더 넓거나 커 보이면 안 된다.
- 긴 일기 때문에 카드 전체가 늘어나는 방식은 거부한다. 메인 화면 높이는 유지하고 내부 스크롤로 처리한다.
- 제목과 내용이 한 줄에 섞여 보이는 이전 구조는 수정한다. 메인 페이지의 다른 카드처럼 제목은 상단, 내용은 아래에 둔다.
- TODO/일정 목록을 가로 pill만으로 유지하는 방식은 수정한다. 스케치처럼 세로 목록으로 읽히는 편이 더 적합하다.

## 6. 내가 직접 해결한 부분과 배운 점
이번 단계에서는 단순히 크기를 키우는 것보다, 카드 내부의 정보 위계를 정리하는 것이 중요했다. 처음에는 긴 일기 텍스트가 카드 안에서 스크롤되는 것처럼 보였지만 실제 브라우저 측정에서 텍스트 높이가 내용만큼 커지는 문제가 확인되었다. 그래서 grid row와 flex 기준을 다시 잡고, 본문 영역에 명시적인 높이와 `overflow: auto`를 적용해 실제 스크롤이 생기도록 직접 수정했다.

## 7. 다음 단계
메인 캘린더 하단 UI를 최종 캡처 후보로 확인한다. 긴 일기, 일정이 많은 날짜, 일정이 없는 날짜를 각각 눌러 카드 높이와 내부 스크롤이 유지되는지 점검한다.
