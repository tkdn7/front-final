# 메인 하단 TODO/일정과 AI 브리핑 대상 시간순 정렬

## 1. 사용자 프롬프트 원문
메인페이지의 캘린더 하단 TODO/일정 항목의 정렬과 AI 페이지의 AI 브리핑 대상 정렬순서는 시간순으로 오름차순 해줘.
메인페이지의 TODO/일정 정렬과 AI 브리핑대상 정렬순서만 수정해야해! 다른 건 건들이지 마!

## 2. AI 답변 요약
- 수정 범위를 메인 캘린더 하단 TODO/일정 목록과 AI 브리핑 대상 목록으로 제한했다.
- AI 브리핑 대상은 이미 시간순 정렬 함수를 사용하고 있어 로직을 유지하고 검증했다.
- 메인 캘린더 하단 TODO/일정 목록은 `sortSchedulesByTime(dayEvents)`를 거친 목록을 렌더링하도록 수정했다.
- 캘린더 점 표시, TODO 페이지 목록 정렬, 저장 데이터 구조 등 다른 기능은 변경하지 않았다.
- 로컬 Chrome 검증에서 메인 하단과 AI 브리핑 대상 모두 08:00, 09:00, 14:00, 21:00 순으로 표시되는 것을 확인했다.

## 3. AI 활용이 필요한 문제 상황
기존 일정 데이터는 우선순위와 완료 여부에 따라 정렬되는 부분이 있어, 사용자가 보는 특정 영역만 시간순으로 바꿔야 했다. 전체 정렬 함수를 바꾸면 TODO 페이지나 캘린더 점 표시 같은 다른 화면까지 바뀔 수 있어 수정 범위를 분리해야 했다.

## 4. 채택한 내용
- 메인 캘린더 하단 렌더링에서만 `displayedDayEvents = sortSchedulesByTime(dayEvents)`를 사용했다.
- AI 브리핑 대상은 기존 `sortSchedulesByTime(...)` 적용 상태를 유지했다.
- 시간 없는 일정은 `99:99`로 취급해 시간 있는 일정 뒤에 오도록 했다.

## 5. 거부하거나 수정한 내용
- `scheduleItemsForDate()` 자체를 시간순으로 바꾸는 방식은 다른 화면까지 영향이 갈 수 있어 제외했다.
- TODO 페이지나 메인 좌측 SCHEDULE 목록 정렬은 사용자가 요청하지 않았으므로 수정하지 않았다.
- 캘린더 점 표시 로직도 이전 수정 상태 그대로 유지했다.

## 6. 내가 직접 해결한 부분과 배운 점
- 같은 일정 데이터라도 표시 영역마다 정렬 기준을 다르게 적용해야 할 수 있다는 점을 확인했다.
- 공통 함수 자체를 바꾸는 것보다, 필요한 화면에서만 별도 표시용 배열을 만들어 렌더링하는 방식이 더 안전했다.
