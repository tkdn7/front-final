# 19 메인캘린더 TODO높이 일기표시수정

## 1. 사용자 프롬프트 원문
1. 메인페이지 캘린더 좌측의 TODO리스트와 동일한 세로길이로 보이도록 수정
2. 한줄일기에 보이는 이모티콘은 일기페이지에서 추가한 이모티콘임
일기가 작성되지 않은 경우 이모티콘도 안 보여야 함.
3. 캘린더 2분할 한줄일기의 세로 길이는 더 작아져도 괜찮음.

## 2. AI 답변 요약
- 메인 화면의 왼쪽 TODO 패널과 가운데 캘린더 패널의 높이가 다르게 보여 균형이 깨지는 문제를 확인했다.
- 캘린더 패널 높이를 왼쪽 TODO 패널과 같은 계산식으로 맞춘다.
- 날짜 선택 후에는 캘린더 내부 공간이 줄어드므로, 날짜칸과 하단 분할 영역을 함께 압축한다.
- 한줄일기가 없는 날짜에서는 기본 이모티콘을 보여주지 않는다.
- 일기가 있는 경우에만 일기 페이지에서 저장한 이모티콘과 텍스트를 함께 보여준다.
- 한줄일기 카드의 세로 비율을 줄이고 TODO/일정 영역을 상대적으로 더 넓게 유지한다.
- 브라우저 검증에서 TODO 패널과 캘린더 패널이 모두 387px로 같고, 일기 없는 날짜의 이모티콘이 0개로 확인됐다.

## 3. AI 활용이 필요한 문제 상황
이전 상태에서는 메인 캘린더가 왼쪽 TODO 리스트보다 훨씬 길게 보여 사이트맵형 대시보드 균형이 맞지 않았다. 또한 일기가 없는 날짜에도 기본 이모티콘이 표시되어, 실제로 사용자가 일기 페이지에서 추가한 감정 이모티콘처럼 오해될 수 있었다.

## 4. 채택한 내용
- `.calendar-panel`의 높이를 왼쪽 2단 스택의 상단 TODO 패널과 같은 `calc((100% - 20px) / 2)` 기준으로 맞췄다.
- 날짜 선택 후에는 `.calendar-panel.split-open` 클래스를 붙여 캘린더 날짜칸과 하단 분할 영역을 압축했다.
- 선택 상태에서는 날짜칸, 점 표시, 하단 카드 padding, 제목 크기를 줄여 같은 높이 안에 들어가도록 조정했다.
- 한줄일기 영역은 일기가 있는 경우에만 저장된 이모티콘을 렌더링한다.
- 일기가 없는 경우에는 이모티콘 없이 안내 문구만 보여주도록 수정했다.
- 하단 2분할에서는 한줄일기보다 TODO/일정 영역이 더 넓게 보이도록 비율을 조정했다.
- 선택 후 한줄일기 카드 42px, TODO/일정 카드 70px로 패널 안에서 넘치지 않게 조정했다.

## 5. 거부하거나 수정한 내용
- 일기가 없을 때 기본 `🙂` 이모티콘을 보여주는 방식은 거부했다. 이모티콘은 일기 페이지에서 사용자가 저장한 값이어야 하기 때문이다.
- 캘린더를 메인 페이지 전체 높이로 계속 늘리는 방식은 수정했다. 왼쪽 TODO 리스트와 같은 세로 길이를 맞추는 것이 최신 요구사항이다.
- 선택 후 하단 2분할을 그대로 큰 높이로 유지하는 방식은 제외했다. 제한된 높이 안에서 TODO/일정 가독성을 유지해야 하기 때문이다.

## 6. 내가 직접 해결한 부분과 배운 점
이번 수정에서는 화면의 전체 균형을 맞추기 위해 단순히 카드 하나를 줄이는 것이 아니라, 선택 전/후 상태를 구분해야 했다. 일기 데이터가 없을 때 기본값을 보여주는 것이 항상 좋은 UX는 아니며, 사용자가 직접 입력한 데이터와 시스템이 임의로 넣은 기본값을 구분해야 한다는 점을 확인했다.

## 7. 다음 단계
실제 화면에서 사용자가 원하는 시각 비율과 맞는지 확인한다. 이후 수정이 필요하면 캘린더 패널 높이는 유지하고, 선택 후 내부 카드 비율만 조정한다.

## 8. 후속 정정
이 단계의 `캘린더 패널 전체 높이를 TODO 패널과 동일하게 줄인다`는 해석은 사용자 의도와 달랐다. 이후 피드백에 따라 캘린더 전체 길이는 유지하고, 캘린더의 위쪽 날짜 영역만 TODO 리스트 세로 길이와 동일하게 보이도록 다시 수정했다.
