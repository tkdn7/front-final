# 메인 SCHEDULE, TODO 날짜 점 표시, 일기 목록 UI 수정

## 1. 사용자 프롬프트 원문
[메인페이지]
1. TODAY TODO 리스트에 오늘의 일정만 보이는 문제가 있음. 아직 미완료된 일정이 모두 표시되도록 수정.
-> 소제목의 경우 TODAY는 SCHEDULE로 바꿔줘. 폰트 등 디자인 적 부분은 바꾸면 안돼.
2. 아직도 한줄일기 이모티콘 잘려 보이는 문제가 있음.
-> 한줄일기와 TODO/일정의 비율이 2:3이 되도록 수정
[TODO]
1. TODO 날짜에 표시되는 미완료 개수 글자 대신 점으로 표시 되도록 수정 (메인페이지의 캘린더에 표시되는 점과 동일하게)
2. TODO 날짜의 요일이 추가되면서 일과 요일이 표시되는 칸에 미완료 개수 글자가 겹치는 문제가 있음
-> 미완료 표시 우측으로 조금 이동
[일기]
1. TODO 날짜와 동일하게 요일과 이모티콘이 겹치는 문제 발생
-> 이모티콘 우측으로 요일과 겹치지 않도록 이동
2. 이모티콘이 너무 작게 보이는 문제가 있음
-> 이모티콘 크기 키우기
3. DIARY에 일기 내용 저장하는 것을 TODO의 일정 추가 버튼 처럼 일기 추가 버튼을 만들어서 작성할 수 있도록 수정
4. DIARY 페이지에는 그동안의 일기를 한눈에 볼 수 있도록 UI 변경 (스크롤을 통해 연속으로 확인 가능)
-> 좌측의 일기 날짜에서 날짜 클릭시 우측의 DIARY에서 해당 날짜의 일기를 확인 가능하도록 스크롤 이동

## 2. AI 답변 요약
- 메인 좌측 TODO 패널의 소제목을 TODAY에서 SCHEDULE로 바꾸되 기존 폰트와 배치 스타일은 유지했다.
- 메인 SCHEDULE 목록은 오늘 일정만 필터링하지 않고, 아직 완료되지 않은 모든 TODO를 보여주도록 수정했다.
- 캘린더 하단 2분할 영역은 한줄일기와 TODO/일정의 세로 비율이 2:3이 되도록 CSS grid 비율을 조정했다.
- TODO 날짜 레일의 미완료 개수 텍스트는 제거하고 캘린더와 같은 우선순위 색상 점으로 바꾸었다.
- 날짜와 요일, 점/이모티콘이 겹치지 않도록 날짜 레일의 열 너비와 오른쪽 여백을 조정했다.
- 일기 페이지는 우측 본문을 작성 폼이 아니라 일기 목록 화면으로 바꾸고, 일기 추가 버튼을 통해 모달에서 작성하도록 수정했다.
- 좌측 일기 날짜를 클릭하면 선택 날짜의 일기 항목이 우측 목록에서 강조되고 해당 위치로 스크롤되도록 연결했다.
- 브라우저 렌더링 기준으로 SCHEDULE 표시, 2:3 비율, 점 표시, 일기 모달 저장 동작을 확인했다.

## 3. AI 활용이 필요한 문제 상황
메인 페이지, TODO 페이지, 일기 페이지가 서로 같은 날짜 UI를 공유하고 있어서 한 부분만 수정하면 다른 화면의 정렬이나 표시 방식이 어긋날 수 있었다. 특히 TODO 날짜 레일의 개수 텍스트를 점으로 바꾸면서 우선순위 색상, 요일 표시, 완료 항목 정렬 규칙을 함께 유지해야 했다.

## 4. 채택한 내용
- 메인 SCHEDULE에는 `pendingTodos()`를 사용해 모든 미완료 일정을 표시하도록 했다.
- 캘린더 하단의 한줄일기와 TODO/일정 카드 비율은 `2fr / 3fr`로 조정했다.
- TODO 날짜 레일은 `event-dot`을 재사용해 메인 캘린더와 같은 방식의 점 표시로 통일했다.
- 일기 날짜 레일은 날짜, 요일, 이모티콘이 분리되어 보이도록 grid column과 summary padding을 넓혔다.
- DIARY 우측 영역은 작성된 일기를 연속 목록으로 보여주고, 일기 추가 버튼을 눌러 모달에서 작성하는 방식으로 변경했다.
- 일기 저장 후 모달을 닫고, 우측 목록에서 저장된 일기가 활성화되도록 했다.

## 5. 거부하거나 수정한 내용
- 미완료 개수를 텍스트로 유지하는 방식은 날짜와 요일이 겹쳐 보일 수 있어 제외했다.
- 일기 목록을 선택한 한 달로만 제한하는 방식은 “그동안의 일기를 한눈에”라는 요구와 맞지 않아 전체 작성 일기 목록으로 수정했다.
- 메인 SCHEDULE 패널의 제목 폰트, progress row, 버튼 스타일은 사용자가 디자인 변경을 원하지 않았기 때문에 구조를 크게 바꾸지 않았다.
- 일기가 없는 날짜에 “미작성” 같은 안내 텍스트를 날짜 레일에 넣는 방식은 이전 요구와 충돌하므로 넣지 않았다.

## 6. 내가 직접 해결한 부분과 배운 점
- 직접 판단한 부분은 TODAY라는 표현이 기능 범위와 맞지 않으므로 SCHEDULE로 바꾸되, 패널의 시각 디자인은 유지한 점이다.
- TODO 날짜의 정보량이 늘어나면 텍스트보다 점 표시가 더 깔끔하다는 점을 확인했다.
- 일기 작성 UI를 본문 안에 항상 노출하는 것보다 버튼과 모달로 분리하는 편이 TODO 페이지의 일정 추가 흐름과 더 통일감이 있었다.
- 브라우저 검증을 통해 실제 2:3 비율이 적용되었고, 이모티콘이 카드 내부에서 잘리지 않는 것을 확인했다.
