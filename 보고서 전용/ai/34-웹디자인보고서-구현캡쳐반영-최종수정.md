작업 중간중간 확인해야할 내용 있으면 물어봐!

## 2. AI 답변 요약
- 기존 보고서 초안 PPTX와 구현결과 폴더의 이미지 목록을 확인했다.
- 구현결과 이미지는 라이트/다크 모드가 짝으로 구성된 18개 PNG였고, 기존 초안은 구현결과 슬라이드가 부족해 일부 이미지만 들어간 상태였다.
- 보고서 흐름은 유지하되 구현결과 파트를 9장으로 확장해 각 화면의 light/dark 캡쳐가 모두 들어가도록 재구성했다.
- 최근 수정사항인 TODO 일정관리 정렬 기준과 옥정 날씨 지역명 보정 내용을 데이터/API, AI 활용, 테스트/수정 슬라이드에 반영했다.
- PPTX 렌더링 후 22개 슬라이드 미리보기와 18개 이미지 삽입 여부, 레이아웃 경계 오류를 검증했다.

## 3. AI 활용이 필요한 문제 상황
보고서 초안은 이미 있었지만, 실제 구현결과 캡쳐가 모두 반영되지 않았고 이후 웹페이지 수정사항도 보고서 문구에 반영해야 했다. 따라서 캡쳐 파일 목록을 빠짐없이 확인하고, 보고서 구성에 맞춰 슬라이드 위치를 다시 정리하는 작업이 필요했다.

## 4. 채택한 내용
- 구현결과 이미지를 화면별로 light/dark 2개씩 묶어 9개 구현결과 슬라이드로 구성했다.
- 메인 기본 화면, 메인 날짜 선택 화면, TODO, TODO 추가 모달, 습관, 일기, 일기 추가 모달, AI, API 모달 이미지를 모두 포함했다.
- TODO 정렬 수정은 `우선순위 > 일정 시간 > 추가 순서`로 보고서에 반영했다.
- 옥정/옥정동 날씨 조회는 `양주시 옥정동` 기준 지역명 보정으로 설명했다.
- OpenRouter API 키 미저장, GPT 5.4 nano 고정, localStorage 저장 구조를 보고서 문구에 유지했다.

## 5. 거부하거나 수정한 내용
- 구현결과 이미지를 한 장에 너무 많이 넣는 방식은 가독성이 떨어져 제외했다.
- 기존 5장 구현결과 슬라이드에 모든 이미지를 억지로 압축하지 않고, 9장으로 확장해 캡쳐가 작아지지 않도록 수정했다.
- 사이트맵 슬라이드의 일부 연결선은 렌더링 검수에서 음수 bbox로 잡혀 QA 오류가 발생했기 때문에 좌표를 양수 기준으로 수정했다.
- 프로젝트 폴더의 기존 파일명에서 확인한 `20240841 정수빈` 정보를 표지에 반영하고, 제출용 파일명 복사본도 함께 만들었다.

## 6. 내가 직접 해결한 부분과 배운 점
보고서에는 단순히 완성 화면을 나열하는 것이 아니라, 각 화면이 어떤 기능을 증명하는지 함께 설명해야 한다는 점을 확인했다. 특히 구현 이후에도 TODO 정렬, 날씨 지역명 보정처럼 작은 기능 수정이 생기면 보고서의 기능요약, API 설계, 테스트 항목에도 함께 반영되어야 한다는 점을 배웠다.
