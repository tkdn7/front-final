# 20 메인캘린더 날짜영역높이정정

## 1. 사용자 프롬프트 원문
하 아니아니 캘린더 날짜 윗 부분만 TODO 리스트 세로 길이랑 동일하게 보이라고 한 거였고, 캘린더 길이 자체를 줄이면 안돼.

## 2. AI 답변 요약
- 이전 수정에서 캘린더 패널 전체 높이를 줄인 것은 사용자 의도와 다르다.
- 캘린더 전체 패널은 다시 메인 대시보드 전체 높이를 사용하도록 되돌린다.
- `CALENDAR` 제목, 요일, 날짜 그리드를 하나의 `calendar-date-block`으로 묶는다.
- 이 위쪽 날짜 영역만 왼쪽 TODO 패널과 같은 세로 길이로 보이게 한다.
- 날짜 선택 후 하단 한줄일기/TODO 영역은 캘린더 패널의 남은 아래 영역을 사용하게 한다.
- 일기가 없을 때 이모티콘을 숨기는 이전 수정은 유지한다.
- 브라우저 좌표 검증에서 TODO 패널은 93~480, 캘린더 위쪽 날짜 영역도 93~480으로 일치했다.

## 3. AI 활용이 필요한 문제 상황
사용자의 의도는 캘린더 전체 카드의 길이를 줄이는 것이 아니라, 캘린더 내부에서 위쪽 날짜 영역만 TODO 리스트와 같은 높이로 보이게 하는 것이었다. 이전 수정은 전체 패널을 줄여 메인 화면의 캘린더 길이를 훼손했기 때문에 정정이 필요했다.

## 4. 채택한 내용
- `index.html`에서 캘린더 제목, 요일, 날짜 그리드를 `calendar-date-block`으로 감쌌다.
- `.calendar-panel`에 적용했던 `height: calc((100% - 20px) / 2)`와 `align-self: start`를 제거했다.
- `.calendar-date-block`만 `calc((100% - 20px) / 2)` 높이 기준을 사용하게 했다.
- 날짜 그리드는 날짜 영역 내부에서 자연스럽게 채워지도록 `flex`와 `grid-auto-rows`를 적용했다.
- 선택 후 하단 2분할은 캘린더 전체 패널의 남은 아래 영역을 사용하도록 유지했다.
- 캘린더 전체 패널 높이는 793px로 유지하고, 위쪽 날짜 영역만 TODO 패널 387px와 시각적으로 일치하게 했다.

## 5. 거부하거나 수정한 내용
- 캘린더 패널 전체 높이를 TODO 리스트와 동일하게 줄이는 방식은 사용자 의도와 다르므로 수정했다.
- 날짜 선택 상태에서 캘린더 날짜칸을 지나치게 압축하던 규칙은 제거했다.
- 한줄일기와 TODO/일정 하단 영역을 없애거나 줄이는 방식은 사용하지 않았다.

## 6. 내가 직접 해결한 부분과 배운 점
이번 수정에서는 사용자의 표현에서 `캘린더`가 전체 패널을 뜻하는지, 내부 날짜 영역을 뜻하는지 정확히 구분해야 했다. 레이아웃 수정은 전체 컨테이너와 내부 섹션을 분리해서 생각해야 하며, 잘못 해석한 수정은 보고서 기록에도 후속 정정으로 남겨야 한다는 점을 확인했다.

## 7. 다음 단계
실제 화면에서 사용자가 보는 비율이 의도와 맞는지 확인한다. 추가 수정이 필요하면 캘린더 전체 패널 높이는 유지한 상태에서 날짜 영역과 하단 2분할 사이의 간격만 조정한다.
