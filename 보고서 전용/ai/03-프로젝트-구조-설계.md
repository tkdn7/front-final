# 03 프로젝트 구조 설계

## 1. 사용자 프롬프트 원문
앞 단계에서 LIFE CYCLE 앱의 화면 범위를 메인, TODO, 습관, 일기/캘린더, AI 브리핑으로 정리했다. 이제 `tkdn7/front-final` 저장소의 실제 제출용 프로젝트 구조를 설계해줘.

현재 저장소에는 다음이 있다.
- `example/Gamified Resume Design`: 참고용 React/Vite 예시 프로젝트
- `icon/`: TODO, 습관, 일기, AI, 라이트/다크 모드 아이콘 PNG
- `보고서 전용/`: 과제 조건, 구상본, 보고서 자료, AI 활용 기록

요구사항은 다음과 같다.
- 실제 제출 앱은 저장소 루트에 만든다.
- `example` 폴더는 참고만 하고 그대로 제출 앱으로 쓰지 않는다.
- React, TypeScript, Tailwind 기반으로 구성한다.
- 사이트맵의 nav와 아이콘을 살릴 수 있게 파일 구조를 잡는다.
- API 키는 코드, `.env`, localStorage에 저장하지 않는 구조로 설계한다.
- 보고서와 AI 기록 파일은 `보고서 전용/`에 그대로 둔다.

출력은 다음 형식으로 해줘.
1. 루트 파일 구조
2. `src/components`에 들어갈 컴포넌트 목록
3. `src/lib`에 들어갈 유틸 함수 목록
4. `src/styles`에 들어갈 스타일 파일
5. 아이콘 처리 방식
6. 수정하면 안 되는 파일/폴더
7. 이후 구현 순서

코드는 아직 작성하지 말고, 구현자가 그대로 따라갈 수 있는 구조 계획만 작성해줘.

## 2. AI 답변 요약
- 루트에 `package.json`, `index.html`, `vite.config.ts`, `tsconfig.json`, `src/`를 둔다.
- `src/components`에는 AppShell, TopNav, Dashboard, TodoPanel, HabitPanel, CalendarPanel, DiaryPanel, AIBriefingPanel, ApiKeyDialog를 둔다.
- `src/lib`에는 날짜 처리, localStorage 저장, OpenWeatherMap 호출, OpenRouter 호출, 샘플 데이터 관리 함수를 둔다.
- 스타일은 Tailwind 전역 CSS와 테마 변수 중심으로 관리한다.
- 기존 `icon/` 파일은 `public/icon/`으로 복사하거나 import 가능한 asset 경로로 사용한다.
- `보고서 전용/`과 `example/`은 삭제하거나 덮어쓰지 않는다.
- 구현 순서는 프로젝트 초기화, 스타일/레이아웃, 로컬 데이터, API 입력, 실제 API 연동, 테스트 순서가 적절하다.

## 3. AI 활용이 필요한 문제 상황
저장소 안에 참고 프로젝트와 보고서 자료가 같이 있어서 실제 제출 앱을 어디에 만들고, 어떤 파일은 건드리지 말아야 하는지 명확히 정해야 했다. 구조를 먼저 정하지 않으면 example을 잘못 수정하거나 보고서 자료를 섞어버릴 위험이 있었다.

## 4. 채택한 내용
- 실제 앱은 저장소 루트에 만든다.
- `example`은 참고만 하고 제출 앱의 구조로 복사하지 않는다.
- React/Tailwind 구조 제안은 참고하되, 실제 제출 안정성을 위해 `index.html`, `styles.css`, `script.js` 기반 정적 웹사이트로 직접 수정한다.
- 컴포넌트 역할은 HTML 섹션과 JavaScript 렌더 함수 단위로 분리한다.
- API 호출 함수와 localStorage 함수는 `script.js` 안에서 역할별 함수로 분리한다.
- 아이콘은 기존 PNG를 재사용한다.

## 5. 거부하거나 수정한 내용
- AI가 `example/Gamified Resume Design` 안에서만 작업하려고 하면 수정한다. 실제 제출 앱은 루트에 있어야 한다.
- AI가 `보고서 전용/`을 삭제하거나 앱 소스 폴더 안으로 옮기려 하면 거부한다.
- AI가 불필요한 대형 UI 라이브러리를 추가하려 하면 거부한다. 과제 구현에 필요한 최소 구조로 간다.
- AI가 API 키를 `.env`로 받는 구조를 만들면 수정한다. 이번 프로젝트는 사용자가 매번 화면에서 입력하는 방식이다.
- AI가 아이콘을 새로 그리거나 외부 아이콘만 쓰려 하면 수정한다. 기존 `icon/*.png`를 우선 사용한다.
- AI가 React/Tailwind만 가능한 구조로 제한하면 수정한다. 구상 초안본에 HTML, CSS, JS 사용도 명시되어 있으므로 정적 HTML 기반 구현을 선택한다.

## 6. 내가 직접 해결한 부분과 배운 점
프로젝트 구조는 기능 구현보다 먼저 결정해야 한다는 점을 배웠다. 특히 제출용 파일, 참고 파일, 보고서 파일을 구분해야 나중에 압축 제출이나 보고서 작성이 쉬워진다. 기존 example이 있어도 그대로 복사하는 대신, 필요한 아이디어만 참고하고 제출용 구조는 따로 만드는 것이 맞다고 판단했다. 또한 React/Tailwind도 사용할 수 있었지만, 제출과 실행을 단순하게 하기 위해 HTML/CSS/JS 정적 구조로 직접 조정했다.

## 7. 다음 단계
정해진 구조에 맞춰 메인 대시보드, nav, 레이아웃, 테마 토글을 먼저 구현한다.

## 8. 실제 진행 시나리오
이 단계의 결과를 바탕으로 실제 파일 생성 작업을 시작한다. 다음 단계에서는 AI에게 “보고서 전용과 example은 보존하고 루트 앱만 구현하라”고 명시하여 원치 않는 파일 변경을 막는다. 실제 구현에서는 `src/` 대신 루트의 `index.html`, `styles.css`, `script.js`로 조정한다.
