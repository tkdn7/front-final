# 17 캘린더 하단영역 일대일비율수정

## 1. 사용자 프롬프트 원문
캘린더 2분할 시, 캘린더와 하단 내용(한줄일기, TODO / 일정)의 비율이 서로 1:1 이어야 함

## 2. AI 답변 요약
- 캘린더 날짜 영역과 하단 선택 날짜 패널이 같은 높이를 갖도록 flex 비율을 조정한다.
- 날짜 셀의 최소 높이를 제거해 캘린더가 고정 크기 때문에 하단 영역을 밀어내지 않게 한다.
- 캘린더 그리드는 남은 높이 안에서 날짜 행을 균등하게 나누도록 `grid-auto-rows`를 적용한다.
- 하단 패널은 기존 내부 구조와 스크롤 기능을 유지하되, 캘린더와 같은 비율로 공간을 나눠 갖게 한다.
- Chrome에서 날짜 클릭 후 캘린더 높이와 하단 패널 높이를 수치로 비교한다.

## 3. AI 활용이 필요한 문제 상황
이전 구현에서는 하단 영역이 남은 공간을 채우는 방식이라, 캘린더 날짜 영역과 하단 내용 영역이 정확히 1:1로 보장되지 않았다. 사용자가 캘린더와 하단 내용의 비율을 명확히 1:1로 지정했기 때문에, 단순히 눈대중으로 맞추는 것이 아니라 실제 브라우저 높이 측정까지 필요했다.

## 4. 채택한 내용
- `.calendar-grid`에 flex 기준을 적용해 캘린더 날짜 영역도 남은 공간을 비율로 나누게 했다.
- `.day-cell`의 `min-height`를 제거하고, 날짜 숫자와 점이 셀 중앙에 오도록 flex 정렬을 적용했다.
- `.calendar-grid`에 `grid-auto-rows: minmax(0, 1fr)`를 적용해 날짜 행들이 주어진 높이 안에서 균등하게 나뉘게 했다.
- `.date-split`은 기존처럼 하단 패널 역할을 유지하되, 캘린더와 같은 높이를 갖도록 flex 비율을 유지했다.
- 실제 측정에서 캘린더 262px, 하단 261px로 거의 동일한 1:1 비율을 확인했다.

## 5. 거부하거나 수정한 내용
- 캘린더 날짜 셀의 기존 최소 높이를 유지하는 방식은 수정했다. 이 값이 남아 있으면 하단 패널과 정확한 1:1 비율을 만들기 어렵다.
- 하단 영역만 키우거나 줄이는 방식은 거부했다. 캘린더와 하단이 함께 남은 공간을 나눠 가져야 한다.
- 눈으로만 확인하고 끝내는 방식은 보류하고, Chrome에서 실제 높이를 측정했다.

## 6. 내가 직접 해결한 부분과 배운 점
비율 문제는 단순히 `height: 50%`를 주는 것보다, 부모가 flex column인지, 고정 헤더와 weekday row가 있는지, 날짜 셀의 최소 높이가 남아 있는지를 함께 봐야 했다. 실제 측정에서 처음에는 캘린더 258px, 하단 265px로 차이가 있었고, 캘린더 flex를 조금 보정해 최종적으로 262px 대 261px로 맞췄다.
