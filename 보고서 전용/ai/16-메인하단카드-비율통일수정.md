# 16 메인하단카드 비율통일수정

## 1. 사용자 프롬프트 원문
전체적으로 비율이 얘네만 커졌어. 다른 요소들 비율과 일치하도록 수정해서 통일성 유지 시켜줘. 그리고, 소제목이 다 진하고 클 필요 없어.

## 2. AI 답변 요약
- 메인 캘린더 하단 카드의 구조는 유지하되 시각적 크기를 전체 UI 기준에 맞춰 낮춘다.
- 카드 모서리를 과하게 둥근 값에서 기존 디자인 토큰인 `var(--radius)`로 맞춘다.
- 한줄일기와 TODO/일정 소제목 크기를 줄이고, 굵기도 900에서 800 수준으로 낮춘다.
- 날짜와 일정 개수 같은 보조 정보는 큰 제목처럼 보이지 않도록 13px muted 텍스트로 조정한다.
- 한줄일기 본문은 13px, 보통 굵기 계열로 낮춰 다른 리스트/메타 텍스트와 어울리게 한다.
- 이모티콘은 너무 큰 46px에서 32px로 줄여 카드 안에서 튀지 않게 한다.
- 내부 스크롤, 한줄일기보다 TODO/일정이 더 큰 비율, 하단 잘림 방지는 유지한다.

## 3. AI 활용이 필요한 문제 상황
이전 수정에서 기능 요구사항은 반영되었지만, 메인 하단 카드만 글자와 이모티콘이 커져 전체 대시보드 안에서 따로 확대된 것처럼 보였다. 특히 소제목과 개수/날짜 정보가 모두 진하고 크게 표시되어, 기존 TODO 리스트나 습관 카드의 조용한 UI와 맞지 않았다.

## 4. 채택한 내용
- `styles.css`에서 메인 하단 카드의 border-radius를 `var(--radius)`로 맞췄다.
- 하단 카드 패딩을 줄여 다른 패널 내부 요소와 밀도를 맞췄다.
- 카드 소제목은 14px, 보조 정보는 13px로 낮췄다.
- 보조 정보에는 muted 색상과 700 굵기를 적용해 제목보다 덜 튀게 했다.
- 한줄일기 본문은 13px, 500 굵기로 낮췄다.
- 한줄일기 이모티콘은 32px로 줄였다.
- 긴 한줄일기 내부 스크롤과 TODO/일정 영역의 더 큰 높이 비율은 유지했다.

## 5. 거부하거나 수정한 내용
- AI가 첨부 스케치의 큰 글자 크기를 그대로 유지하자고 하면 수정한다. 실제 앱에서는 전체 UI 스케일과 맞아야 한다.
- 소제목, 날짜, 일정 개수를 모두 굵고 크게 표시하는 방식은 거부한다. 정보 위계가 무너져 보인다.
- 이모티콘을 계속 크게 유지하는 방식도 수정한다. 강조는 필요하지만 카드 전체보다 튀면 안 된다.
- 카드 모서리만 과하게 둥근 스타일은 기존 UI와 달라 `var(--radius)`로 통일한다.

## 6. 내가 직접 해결한 부분과 배운 점
이번 단계에서는 이미지 참고를 그대로 확대해서 따라가는 것보다, 이미 만들어진 앱의 디자인 언어와 맞추는 것이 중요했다. 같은 기능이라도 주변 카드, 리스트, 버튼의 크기와 어울리지 않으면 완성도가 낮아 보인다는 점을 확인했다. 그래서 스케치의 구조는 유지하되 실제 구현에서는 크기, 굵기, 모서리, 패딩을 앱 전체 기준으로 다시 조정했다.

## 7. 다음 단계
메인 페이지 전체를 다시 보면서 하단 카드가 캘린더, TODO 리스트, 습관, AI 브리핑 패널과 같은 스케일로 보이는지 최종 확인한다.
