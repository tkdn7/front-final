# 일기 추가 버튼 우측 정렬 수정

## 1. 사용자 프롬프트 원문
[일기]
1. 일기 추가 버튼 끝나는 우측 위치와 일기 내용 박스 끝나는 우측 위치가 서로 동일해져야해.
-> 일기 추가 버튼 위치를 일기 내용 박스 끝나는 우측 위치와 동일해지도록 좌측으로 조금 이동 시켜줘.

## 2. AI 답변 요약
- 일기 페이지의 `일기 추가` 버튼이 일기 내용 박스보다 오른쪽으로 더 나가 보이는 문제를 확인했다.
- 일기 목록은 스크롤 여백 때문에 실제 내용 박스의 오른쪽 끝이 패널 끝보다 안쪽에 위치했다.
- 다른 페이지 버튼에는 영향을 주지 않도록 `#openDiaryDialog` 버튼에만 오른쪽 margin을 적용했다.
- 처음 12px 조정 후 브라우저 좌표 검증에서 8px 차이가 남아 20px로 재조정했다.
- 최종 검증에서 버튼 오른쪽과 일기 내용 박스 오른쪽 좌표가 모두 1377px로 일치했다.

## 3. AI 활용이 필요한 문제 상황
겉보기 정렬 문제는 CSS 코드만 보고 정확히 판단하기 어려웠다. 스크롤 여백과 실제 렌더링 좌표가 영향을 주기 때문에 브라우저에서 버튼과 일기 내용 박스의 오른쪽 좌표를 직접 확인할 필요가 있었다.

## 4. 채택한 내용
- `#openDiaryDialog`에만 `margin-right: 20px`을 적용해 버튼을 왼쪽으로 이동했다.
- 일기 페이지 외 다른 버튼이나 공통 `.panel-heading` 스타일은 건드리지 않았다.
- 로컬 Chrome 렌더링 기준으로 버튼 오른쪽과 일기 내용 박스 오른쪽 좌표가 같은지 확인했다.

## 5. 거부하거나 수정한 내용
- 공통 버튼 스타일이나 전체 패널 헤더 스타일을 수정하는 방식은 다른 페이지 정렬까지 바뀔 수 있어 제외했다.
- 처음 적용한 `12px` 조정은 실제 검증에서 8px 차이가 남아 최종적으로 `20px`로 수정했다.

## 6. 내가 직접 해결한 부분과 배운 점
- 단순히 눈대중으로 간격을 맞추기보다 실제 브라우저 좌표로 확인하면 더 정확하게 UI를 정렬할 수 있다는 점을 확인했다.
- 스크롤이 있는 목록은 `padding-right`와 `scrollbar-gutter` 때문에 눈에 보이는 박스 끝 위치가 일반 패널 끝과 달라질 수 있음을 배웠다.
