# 04 메인 대시보드 구현

## 1. 사용자 프롬프트 원문
이제 LIFE CYCLE 앱의 메인 대시보드를 구현하려고 한다. 이전 단계에서 정한 구조를 유지해서, 사이트맵에 보이는 메인 화면을 React 컴포넌트로 구현해줘.

반드시 지켜야 할 조건은 다음과 같다.
- 저장소 루트의 제출용 앱에서 작업한다.
- `example`과 `보고서 전용` 자료는 건드리지 않는다.
- 상단 nav에는 LIFE CYCLE 로고, TODO, 습관, 일기, AI, 라이트/다크 토글을 둔다.
- 로고 클릭 시 메인 대시보드로 돌아오게 한다.
- 기존 `icon/`의 PNG를 nav 아이콘으로 사용한다.
- 메인 화면 왼쪽에는 TODO 요약과 습관 요약을 둔다.
- 메인 화면 가운데에는 월간 캘린더를 둔다.
- 메인 화면 오른쪽에는 AI 브리핑 패널을 둔다.
- 일정이 있는 날짜에는 일정 개수만큼 작은 점을 표시한다.
- 날짜를 클릭하면 해당 날짜 일정과 한 줄 일기를 캘린더 아래에 보여준다.
- TODO와 습관 목록은 항목이 많으면 내부 스크롤되게 한다.
- 모바일에서는 한 줄 세로 레이아웃으로 자연스럽게 바뀌게 한다.

이번 단계에서는 API 실제 연동까지 하지 말고, API 브리핑 패널은 이후 연동할 수 있는 UI 상태만 만들어줘.

## 2. AI 답변 요약
- `AppShell` 또는 `TopNav`로 전체 레이아웃과 nav를 구성한다.
- 메인 화면은 좌측 요약 영역, 중앙 캘린더 영역, 우측 AI 브리핑 영역으로 나눈다.
- 캘린더에는 월 이동 버튼, 요일 헤더, 날짜 셀, 일정 점 표시, 오늘 날짜 강조를 넣는다.
- 날짜 선택 상태를 두어 선택된 날짜의 일정과 한 줄 일기 영역을 표시한다.
- TODO/습관 패널에는 진행률과 스크롤 가능한 목록을 넣는다.
- AI 브리핑 패널에는 일정 선택 전/후에 다른 안내가 나올 수 있도록 상태를 준비한다.
- 라이트/다크 모드는 CSS 변수 또는 class 토글로 처리한다.

## 3. AI 활용이 필요한 문제 상황
메인 화면은 여러 기능이 동시에 들어가는 가장 중요한 화면이라 배치가 어긋나면 전체 앱의 인상이 무너질 수 있었다. 사이트맵과 같은 정보 구조를 유지하면서 실제 브라우저에서 사용 가능한 UI로 바꾸는 도움이 필요했다.

## 4. 채택한 내용
- 메인 화면을 3열 대시보드로 구성한다.
- nav를 모든 화면의 공통 상단 영역으로 만든다.
- 캘린더 날짜 클릭 시 일정과 일기 영역이 열리도록 한다.
- AI 브리핑 패널은 실제 API 연결 전에도 UI가 보이도록 만든다.
- 반응형에서는 3열을 세로 배치로 전환한다.
- React 컴포넌트 구조는 HTML 섹션과 JavaScript 렌더 함수로 직접 변환해 구현한다.

## 5. 거부하거나 수정한 내용
- AI가 메인 화면을 랜딩 페이지처럼 큰 홍보 문구 중심으로 만들면 수정한다. 이 앱은 실제 도구형 대시보드여야 한다.
- AI가 사이트맵에 없는 큰 배너, 소개 섹션, 마케팅 문구를 추가하면 제거한다.
- AI가 카드 안에 또 카드를 과하게 중첩하면 수정한다. 패널은 기능 단위로만 나눈다.
- AI가 캘린더 클릭 후 일기 영역을 누락하면 수정한다. 이 기능은 사이트맵 핵심이다.
- AI가 아이콘 대신 텍스트 nav만 만들면 수정한다. 기존 PNG 아이콘을 사용해야 한다.
- AI가 React 컴포넌트 파일 생성만 전제로 하면 수정한다. 실제 구현은 `index.html`, `styles.css`, `script.js`에서 같은 역할을 수행하도록 한다.

## 6. 내가 직접 해결한 부분과 배운 점
메인 화면은 보기 좋은 것보다 사용자가 오늘의 할 일, 습관, 일정, AI 조언을 한 번에 확인할 수 있는지가 중요하다는 점을 배웠다. 사이트맵의 화면 비율을 참고하되, 실제 구현에서는 반응형과 스크롤을 고려해야 한다고 판단했다. React 대신 정적 HTML 구조를 사용하면서도 nav, 패널, 캘린더, 모달의 역할은 컴포넌트처럼 나누어 유지했다.
