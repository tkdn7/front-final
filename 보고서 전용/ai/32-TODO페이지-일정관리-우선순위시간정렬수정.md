# TODO 페이지 일정관리 정렬 수정

## 1. 사용자 프롬프트 원문
웹페이지 1가지만 더 수정하자.
[TODO] 일정관리 항목 정렬 순서가 우선순위 > 일정 시간 > 추가한 순서 기준이 적용되어 있지 않아. 현재 우선순위가 가장 위에 정렬되지만, 일정 시간 보다 추가한 순서가 먼저 고려되는 문제가 있어.
-> TODO 페이지의 일정관리 항목 정렬 순서 우선순위 > 일정 시간 > 추가한 순서 로 고려되도록 수정해줘.

## 2. AI 답변 요약
- 기존 `sortTodos()`는 메인 SCHEDULE과 TODO 페이지가 함께 사용하고 있어 전체 정렬 기준을 바꾸면 다른 화면에 영향을 줄 수 있었다.
- 메인 SCHEDULE 정렬은 유지하고, TODO 페이지 우측 일정관리 목록만 별도 정렬 함수를 쓰도록 분리했다.
- `sortTodosForTodoPage()`를 추가해 완료 여부를 먼저 구분한 뒤, 미완료 항목 내부에서 우선순위 > 일정 시간 > 추가한 순서로 정렬했다.
- `renderTodos()`의 `selectedItems`에만 새 정렬 함수를 적용했다.
- 로컬 Chrome 검증으로 높은 우선순위 안에서 09:00이 15:00보다 먼저 나오고, 같은 시간은 추가 순서대로 유지되는 것을 확인했다.

## 3. AI 활용이 필요한 문제 상황
TODO 페이지와 메인 페이지가 같은 TODO 데이터를 사용하지만, 두 화면의 정렬 목적이 달랐다. 전체 정렬 함수를 직접 바꾸면 이전에 맞춰둔 메인 SCHEDULE 정렬까지 바뀔 수 있어, TODO 페이지 일정관리 영역에만 적용되는 정렬 함수를 별도로 만들어야 했다.

## 4. 채택한 내용
- `sortTodosForTodoPage()`를 새로 추가했다.
- 정렬 기준은 완료 여부 > 우선순위 > 일정 시간 > 추가 순서로 적용했다.
- 완료 항목을 아래로 보내는 기존 요구사항은 유지했다.
- `dashboardItems`는 기존 `sortTodos()`를 계속 사용하게 두었다.
- TODO 페이지 우측 `selectedItems`에만 새 정렬을 적용했다.

## 5. 거부하거나 수정한 내용
- 기존 `sortTodos()` 자체를 바꾸는 방식은 메인 화면과 AI 요약 등에 영향을 줄 수 있어 제외했다.
- 날짜 기준 정렬은 TODO 페이지의 선택 날짜 내부 목록에서는 우선순위와 시간 기준보다 덜 중요하므로 적용하지 않았다.
- 완료 항목을 시간순에 섞는 방식은 이전 완료 항목 하단 배치 요구와 충돌하므로 유지하지 않았다.

## 6. 내가 직접 해결한 부분과 배운 점
- 같은 데이터라도 화면마다 요구하는 정렬 기준이 다르면 공통 함수 하나로 처리하기 어렵다는 점을 확인했다.
- 수정 범위를 작게 잡기 위해 새 함수를 만들고 특정 렌더링 위치에만 적용하는 방식이 안전하다는 점을 배웠다.
- 정렬 검증은 우선순위, 시간, 추가 순서가 서로 충돌하는 테스트 데이터를 넣어 확인해야 정확하다.

## 7. 다음 단계
보고서 최종 정리 시 TODO 페이지는 날짜별 일정관리 목록이 우선순위 > 일정 시간 > 추가 순서 기준으로 표시된다고 설명한다.
