# TODO 정렬, 일기 모달 날짜 선택, 지역 표시 수정

## 1. 사용자 프롬프트 원문
[메인페이지]
1. TODO 리스트 항목 정렬 순서는 날짜 > 우선순위 > 추가된 순서 로 정렬되도록 수정
2. TODO 리스트의 완료된 정도 보여주는 표시바는 이제 의미가 없으니까 제거해줘. 대신 미완료 개수는 남겨둬야 해.
3. 캘린더 2분할 TODO/일정의 -개 일정은 미완료 일정 몇개인지 표시되도록 수정할거야.
-> -개 일정을 미완료 -개 로 표시되도록 수정
[일기]
1. 일기 추가 버튼 클릭 후 뜨는 모달페이지에서 날짜 선택할 수 없는 문제가 있어.
-> 일기 추가 모달페이지에서 날짜 변경 가능하도록 수정
2. 일기 추가 모달페이지에서 한줄일기 작성공간 더 크게 키워줘
3. 일기 추가 시 이모티콘 없이 일기저장한 경우 기본 이모티콘이 들어가는 문제가 있어.
-> 이모티콘이 비어있는 경우 " 이모티콘을 선택하세요! " 경고창 띄워줘.
[메인페이지 및 세부 페이지 모두 적용]
1. TODO에 지역을 추가해도 보이지 않는 문제가 있어.
-> 지역이 있는 경우 지역 확인 가능하도록 지역도 추가적으로 표시되게 수정해줘.

## 2. AI 답변 요약
- 메인 TODO 목록 정렬 기준을 날짜, 우선순위, 추가된 순서로 수정했다.
- 기존 완료율 표시바는 현재 기능 흐름에서 의미가 약해져 제거하고, 미완료 개수 텍스트만 남겼다.
- 캘린더 2분할의 TODO/일정 헤더는 전체 일정 수가 아니라 미완료 일정 수를 표시하도록 바꾸었다.
- 일기 추가 모달에 날짜 선택 input을 추가해 모달 안에서 작성 날짜를 변경할 수 있게 했다.
- 한 줄 일기 textarea 높이를 키워 긴 내용을 입력해도 답답하지 않게 했다.
- 이모티콘이 비어 있으면 기본값을 자동 저장하지 않고 경고창을 띄우도록 했다.
- TODO 지역 정보는 메인 목록, 세부 TODO 목록, 캘린더 하단 일정 버튼에서 함께 보이도록 했다.
- 로컬 Chrome 브라우저 검증으로 정렬, 지역 표시, 날짜 변경 저장, 경고창 동작을 확인했다.

## 3. AI 활용이 필요한 문제 상황
이 수정은 단순 문구 변경이 아니라 정렬 기준, localStorage 데이터 표시, 일기 모달 저장 흐름이 함께 연결되어 있었다. 특히 날짜 변경 후 저장되는 일기 날짜와 지역 표시가 메인/세부/캘린더 하단에서 모두 일관되게 보여야 했기 때문에 AI에게 전체 연결 구조를 점검하게 했다.

## 4. 채택한 내용
- TODO 정렬은 완료 항목을 아래로 보내는 기존 규칙을 유지하면서, 미완료 항목 내부에서 날짜 > 우선순위 > 추가된 순서로 정렬했다.
- 메인 TODO 패널에서는 진행률 바를 제거하고 `미완료 n개` 텍스트만 남겼다.
- 캘린더 하단 TODO/일정 영역은 선택 날짜의 미완료 일정 개수를 `미완료 n개`로 표시했다.
- 일기 모달에 `date` input을 추가하고 날짜 변경 시 선택 날짜와 일기 목록, 캘린더 선택 상태가 같이 바뀌도록 했다.
- 이모티콘 없이 저장하려는 경우 `이모티콘을 선택하세요!` 경고창을 띄우고 저장을 중단했다.
- 지역이 있는 TODO는 메타 정보에 `지역 ...` 형태로 표시하도록 했다.

## 5. 거부하거나 수정한 내용
- 완료율 progress bar를 다른 의미로 재사용하는 방식은 사용자가 제거를 요청했으므로 채택하지 않았다.
- 이모티콘 입력이 비어 있을 때 기본 이모티콘을 자동으로 넣는 기존 방식은 사용자 요구와 맞지 않아 제거했다.
- 캘린더 하단의 일정 개수는 전체 일정 수를 보여주면 완료 항목까지 포함되어 혼동될 수 있어 미완료 기준으로 수정했다.
- 지역을 별도 큰 버튼이나 새 레이아웃으로 분리하는 방식은 UI가 복잡해질 수 있어 기존 메타 정보와 일정 버튼 내부에 자연스럽게 추가했다.

## 6. 내가 직접 해결한 부분과 배운 점
- 사용자가 원하는 정렬 기준이 실제 데이터에서는 완료 여부와 함께 적용되어야 한다는 점을 확인했다.
- 기능이 바뀌면 기존 UI 요소가 더 이상 의미 없을 수 있으므로, 무조건 남기기보다 제거하는 판단도 필요하다는 점을 배웠다.
- 모달 내부 날짜 변경처럼 작은 입력 흐름도 저장 대상 날짜, 캘린더 선택 날짜, 목록 강조 상태와 함께 움직여야 사용자가 헷갈리지 않는다.
- 기본값 자동 입력은 편리해 보일 수 있지만, 보고서/과제 조건상 사용자의 직접 입력 의도를 흐릴 수 있어 경고 방식이 더 적절했다.
